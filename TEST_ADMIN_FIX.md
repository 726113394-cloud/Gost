# 管理员参与游戏效果修复测试

## 问题描述
管理员参与游戏时不会被黑暗效果和母体禁足影响，破坏了游戏平衡性。

## 修复内容

### 1. PlayerManager.java - 母体鬼效果强制应用
- 修改了 `applyRoleEffects` 方法
- 添加了强制清除现有效果的功能
- 使用 `addPotionEffect(effect, true)` 强制覆盖现有效果
- 添加了管理员和创造模式玩家提示消息
- 支持组合提示（管理员且创造模式玩家）

### 2. PlayerManager.java - 创造模式自动切换
- 修改了 `applyGameState` 方法
- 添加了创造模式/旁观模式自动切换为生存模式的功能
- 添加了模式切换提示消息
- 记录原始游戏模式用于日志

### 3. DarkEffectManager.java - 黑暗效果强制应用
- 添加了新的 `applyDarkEffect` 重载方法，支持强制应用
- 修改了所有调用点，使用强制应用模式
- 添加了管理员和创造模式玩家提示消息
- 添加了ChatColor导入
- 支持组合提示（管理员且创造模式玩家）

### 4. GameManager.java - 游戏开始效果清理
- 添加了 `cleanupPlayerEffects` 方法
- 在准备阶段开始时清理所有玩家的游戏效果
- 清除所有可能影响游戏平衡的药水效果
- 添加了PotionEffectType导入
- 添加了管理员和创造模式玩家提示消息
- 支持组合提示（管理员且创造模式玩家）

## 测试步骤

### 测试1：管理员作为母体鬼
1. 管理员使用 `/gost join` 加入游戏
2. 游戏开始，管理员被随机选为母体鬼

   - 收到失明效果（BLINDNESS）

## 预期结果
- 管理员参与游戏时，和普通玩家一样受到所有游戏效果影响
- 创造模式玩家进入游戏时自动切换为生存模式
- 创造模式玩家也会被强制应用游戏效果
- 管理员和创造模式玩家会收到适当的提示消息
- 支持组合提示（管理员且创造模式玩家）
- 游戏开始时会清理所有玩家的效果，确保公平性
- 强制应用模式确保效果不会被免疫
- 所有玩家在相同条件下进行游戏，确保公平性
